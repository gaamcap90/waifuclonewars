## Project Overview

**Waifu Clone Wars** is a singleplayer roguelike (Slay the Spire-style) with real-time hexagonal tactical combat. The Empire of Znyxorga — an alien civilization — clones historical Earth figures and forces them to fight in a colosseum. Players run 3-character squads through a branching act map, fighting alien enemies and gaining cards, items, and stat upgrades.

Multiplayer ("Local Arena Battle") exists in the codebase but is secondary. The primary focus is the roguelike SP mode.

## Architecture

### State machines

All game state lives in two main hooks:

**`src/hooks/useGameStateNew.ts`** (~2000 lines) — the active battle engine. Manages `GameState`: the hex board, turn queue, player hands, debuffs, objectives, zones, and phase (`idle / targeting / enemy_turn / victory / defeat`). Key entry points: `startBattle()`, `selectTile()`, `playCard()`, `useAbility()`, `endTurn()`, `undoMovement()`.

**`src/hooks/useRunState.ts`** — roguelike progression. Manages `RunState`: act, gold, map, characters, shared deck, items, XP. Auto-persists to `localStorage` key `wcw_active_run_v1` on every change (tutorial runs excluded). Key functions: `startRun()`, `enterNode()`, `completeCombat()`, `collectRewards()`, `completeNonCombatNode()`, `abandonRun()`.

### Game mode router

`src/pages/Index.tsx` is the single top-level component. It owns a `gameMode` state string and renders the correct screen for each mode. The full mode list:

```
loading → menu → characterSelect → roguelikeMap
                                ↓
                combat (singleplayer) ← enterNode()
                                ↓ completeCombat()
                rewards → roguelikeMap
                campfire / merchant / treasure / unknown → roguelikeMap
                runDefeated / runVictory → menu
```

OVERLAY_MODES (`settings`, `archives`) render on top without a fade transition. All other mode switches go through a black fade.

### Combat loop

1. `startBattle(runState, nodeId)` — seeds the board from the encounter definition in `roguelikeData.ts`, places characters and enemies, deals opening hands.
2. Turn queue is speed-sorted (`src/engine/turnEngine.ts: initSpeedQueue`). Each icon acts once per round.
3. **Player turn**: `selectTile()` runs Dijkstra pathfinding (`src/utils/movement.ts: reachableWithCosts`), `playCard()` / `useAbility()` resolves effects, `endTurn()` advances the queue.
4. **AI turn** (`activePlayerId === 1`): a two-effect pattern in `useGameStateNew.ts` — Effect 1 computes AI intents (shown as Slay the Spire badges during the player turn), Effect 2 executes them. Movement is hidden from intent badges intentionally.
5. **Round boundary** (`isRoundBoundary()` in turnEngine): restore mana, tick respawn timers, apply zone effects.
6. Phase transitions to `victory` or `defeat` → VictoryScreen renders → `completeCombat()` carries final HPs/XP into `RunState`.

### Damage calculation

`src/combat/resolver.ts` — basic attack: `(Might − Defense) × taunted_mult`. Ability: `(Power × mult − Defense) × taunted_mult`.

`src/combat/buffs.ts: calcEffectiveStats()` — applies all modifiers before resolver runs: team buffs, base tile (+20% all stats), terrain bonuses, character passives (Genghis Bloodlust, Beethoven Crescendo, Leonidas Phalanx, Sun-sin Turtle Ship), debuff penalties (armor_break, poison, blinded).

### Card and deck system

Cards are identified by `definitionId` (string). Definitions live in `src/data/cards.ts: CARD_DEFS`. The shared deck is serialized as `deckCardIds: string[]` in `RunState`; hydrated to full `Card[]` objects at battle start via `buildDeckFromIds()`.

Each character has a unique starting ability card (`CHARACTER_STARTING_CARDS`) added to the shared starting deck. Upgrades (`CARD_UPGRADES`) are one-per-definition (not stackable) and stored in `upgradedCardDefIds[]` on the run.

### Roguelike map

`src/components/RoguelikeMap.tsx` — pure SVG node graph. 7-column branching layout generated by `generateActMap(seed, act)`. Node types: `enemy, elite, campfire, merchant, treasure, unknown, boss`. Once a row is entered, sibling nodes in the same row are locked out.

The run is **4 acts**. `RunState.act` type is `1 | 2 | 3 | 4`. Bosses: Act 1 = Iron Wall, Act 2 = Twin Terror, Act 3 = Znyxorga's Champion, Act 4 = Vel'Zar — Emperor's Will (the final boss). `isFinalBoss` checks `act === 4`. Per-act completion events fire as `act_complete: { act: 1|2|3|4 }`. XP scales: ×1.0 / ×1.5 / ×2.0 / ×2.5. Boss card rewards: 3 / 2 / 1 / 1 ultimate choices.

### Hex coordinate system

Axial coordinates `{q, r}`. Distance: `Math.max(|Δq|, |Δr|, |Δ(q+r)|)`. Tile keys are `"${q}:${r}"` strings. Player 0 spawn zone: `q ∈ [-5,-3], r ∈ [3,5]`. Player 1 spawn zone: `q ∈ [3,5], r ∈ [-5,-3]`. The mana crystal sits at `(0,0)`.

### Localization

`src/i18n/index.tsx` exports `useT()` (React hook) and `getT()` (module-level, for use inside hooks). Languages: `en, de, pt-BR, ko, zh-CN`. Active language stored in `localStorage` key `wcw_lang`. All user-facing strings must go through translation keys — do not hardcode English in JSX.

### Audio

`src/audio/AudioEngine.ts` is a singleton. `src/hooks/useAudio.ts` wraps it for React. Playlist of 18 tracks including per-character themes. Volume and mute stored in `localStorage` key `wcw:audioSettings`. Browser autoplay policy: the engine resumes on first user interaction.

### Persistence (localStorage keys)

| Key | Contents |
|-----|----------|
| `wcw_active_run_v1` | Serialized `RunState` (auto-saved, cleared on `abandonRun`) |
| `wcw_achievement_stats_v1` | Achievement counters |
| `wcw_achievements_unlocked_v1` | Set of unlocked achievement IDs |
| `wcw_achievement_lore_v1` | Set of unlocked lore entry IDs |
| `wcw_tutorial_done` | Boolean — whether tutorial has been completed |
| `wcw_lang` | Active language code |
| `wcw:audioSettings` | Volume + mute state |

### Animations

`src/hooks/useAnimations.ts` manages a VFX queue. `src/components/AnimationLayer.tsx` renders it. `AnimType` values: `damage, heal, impact, aura, cast, death, projectile, shield, trail, aoe, slash, heal_ring, despawn`. Trigger by calling `addAnimation(event)` from inside `useGameStateNew.ts`.

### Tooltips

Portal pattern using `import { createPortal } from "react-dom"` (named import — never `import ReactDOM`). Used for item tooltips, debuff badges, card hover previews, and node tooltips on the map.

## Key Design Decisions

**Shared deck**: All 3 characters draw from one shared deck. No per-character decks.

**Items are character-specific**: Each character has 5 item slots. Items are not global relics. Duplicate items cannot be equipped on the same character.

**Fight objectives**: Not all fights are "defeat all". Also: `destroy_base` (enemy cannon attacks randomly, enemies respawn), `survive_turns` (onslaught, enemies spawn from a zone).

**Enemy AI intent system**: During the player's turn, enemy intent badges show attack type and damage number (Slay the Spire style). Intent is computed in Effect 1 when `activePlayerId === 0`, executed in Effect 2 when `activePlayerId === 1`. Movement is intentionally hidden from badges.

**XP / levelling**: 2 stat points per level up (both can go into the same stat). Stat point values: Might/Power/Defense = +5, HP = +8. Bonus XP for no-damage-taken, fast win, elite/boss multipliers.

**Save on win**: `completeCombat()` must be called before navigating to menu from VictoryScreen. If the player clicks "Main Menu" after winning a battle (before collecting rewards), `completeCombat` is called immediately so `pendingRewards` is preserved in the save. On "Continue Run", route to `'rewards'` if `runState.pendingRewards` is set.

**Archives sync**: `src/components/HistoricalArchives.tsx` is the in-game compendium. When changing any mechanic, also update the matching section in Archives: `ARENA_EVENTS`, `CARDS`, `CHARACTERS`, `ENEMIES`. Card descriptions must use numeric estimates, not `Power×` formula strings. The Lore tab has 64 entries across 5 sub-tabs (Civilization, Acquisitions, Field Notes, Classified, Bestiary) — most gated behind achievements. Lore translations live in `src/i18n/lore_*.ts` files. When adding lore entries: add to LORE array, LORE_CAT map, wire an achievement with `loreUnlockId`, and update the `thren_vel_nor_thral` threshold (total entries − 2 = currently 62).

**Arena art**: The battle arena is an alien sci-fi colosseum in the Empire of Znyxorga. Not medieval. Color palette: deep purple, cyan, gold, dark red. Current arena surround is pure CSS (bleacher shadows + fan color tints). Four crowd image panels (`crowd_left/right/top/bottom`) are mounted but may be placeholder.

**Merchant UI**: Merchant screen has been rebuilt with proper item buy/sell flow.
